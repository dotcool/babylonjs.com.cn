      babylonjs摄像机

      在任意的babylonjs的场景中，你可以创建很多摄像机，但是同时只能有一台摄像机是激活状态(除非你使用了multi-viewports).

      摄像机的管理是很简单的，babylonjs提供了很多种类的摄像机，我们将从最常见的两种(自由摄像机和轨迹旋转摄像机)

      • 自由摄像机 - 这是一个第一视角种类的摄像机，当你使用鼠标和键盘来控制时，和很多摄像机一样，你也可以添加控制键比如wasd等
        自由相机并不会自动对准目标，但是在构建自由摄像机时，你可以轻易的设置他锁定到某一个纹理或者向量的位置使用lockedtarget属性。这里示例了你如何构造一个自由摄像机：

      自由摄像机初始化时被构造为z正方向，点的数值就是上边构造函数中写入的，放置在一个有用的开始位置。自由摄像机有很多属性你可以用来适应到你的视图，这里有一些常用的属性position, rotation, speed, inertia, and fov. 自由摄像机也被用作很多其他种类的摄像机 查看更多信息FreeCamera API page .

      • 轨迹旋转摄像机 - 这是一种围绕着目标锚点旋转的摄像机，他可以被鼠标的焦点或者是点击、触摸时间控制，同时他需要第三方的文件hand.js。

      轨迹旋转摄像机用起来有点起卦，但是你会经常使用到，他们有3个独特的属性分别是 alpha (in radians), beta (in radians), and radius (a number). 如果你想像一下轨迹旋转摄像机是一个围绕地球的卫星那么alpha就是纵向或横向轴线, beta就是纬度或上/下轴,而radius 就是地球的半径

      轨迹旋转摄像机有很多属性用来适应你的师徒，最常见是的 alpha, beta, radius, target, speed, inertia, and fov.
      默认情况下，没有设定alpha和beta的值，轨迹旋转摄像机是指向x轴正方向上，具有讽刺意味的是，没有旋转的属性只有位移的属性，因为轨迹旋转摄像机的方向是相对他的目标target属性的，所以可以通过_setPosition方法来设定位置。

      在下边的这个离职，我们将会构建一个轨迹旋转摄像机有一个Zero()的对象没有初始化 alpha, beta, or radius 这些的值. (提示：在没有使用_setPosition设定位置的时候轨迹旋转摄像的朝向有些奇怪，通过Vector3形式的向量我们可以设定这些属性的值

      当我们使用了有用的setPosition方法，我们并不在意我们自己的_alpha, beta, and radius属性。我们只是要确认设置了 target property ( 我们通过new BABYLON.Vector3.Zero() 方法设定的),然后使用方法setPosition() 将摄像机放置到我们需要的3D空间中. 方法setPosition()很容易使用。

      特殊用途摄像机

      在babylonjs中有些特殊用途的摄像机，我们将通过测试他们来解释

      • 触摸摄像机 - 触摸摄像机的工作方式和hand.js很相似，是DOM手势时间的一种现代技术。触摸是一种手势，不论是使用手指还是脚趾在pad还是屏幕上触摸所有的事件都会被触发。这就被识别为手势，你可以通过访问这里了解更多信息 a wikipedia page that tells all about it.

      触摸摄像机对于几乎所有的现代手势输入都是很特别的。大部分是由于hand.js提供了支持，如果你喜欢学习hand.js和他的手势方法，请查看 this comprehensive blog entry

      触摸摄像机有很多魔法方法，像很多来自babylon里一样，我们将让他变得容易很舒适，这里展示了使用触摸摄像机：

      触摸摄像机以自由摄像机为基础，所以有用的特性和方法都和我们的自由摄像机接近，都可以在触摸摄像机中找到。你可以浏览这些属性和方法在触摸摄像机中 our API documentation site.

      • 装置方向摄像机 - 装置方向摄像师是专门设计用来和装置的方向事件交互。当你前后、左右移动你的移动设备都会引发对摄像机的控制或者是场景中的单位，更多信息a wonderfully-detailed blog entry
